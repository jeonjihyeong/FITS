이중로그인 방지 기능
유저가 다른 아이피에서 접글할떄 차단하는 기능

-redis통해서 사용할 예정
배포를 안하면 사실상 의미없는 기능
-레디스에 아이디를 키로 지정하여 value에 현재 접속중인 ip를 지정한다. 
-> 로그인 요청이 들어왔을때 ip가 같다 = 이미로그인중입니다.
-> 로그인 요청이 들어왔을때 ip가 다르다 = 강제로그아웃을 시키고 새로운 로그인을 함
-> 로그인 요청이 들어왔을때 ip에 지정된것이 없다. = 로그인을 한다.

로그아웃시 
서버로 요청을 보내서 redis에 해당 아이디에 있는 value를 del한다.

강제로그아웃 시키기
ip가 달라서 강제로그아웃을 해야하는 경우 => redis 에 해당 value 와 리프레쉬 토큰을 삭제한다.
=>프론트로 에러를 던지거나 message를 던진다. => 그 메시지를 받으면 프론트 anonymous레이어에서 
로그아웃로직을 실행한다.(토큰 드랍, state 바꾸기, 라우팅 메인으로) 

회원가입시
회원가입요청이 들어왔을때 회원가입전 Validation->회원가입이 가능하면 postgresql에 회원정보 저장.
->redis에 아이디값을 추가해놓고 뒤에 value를 (nil)로 바꾸어 놓는다. ->근데 만약 postgre는 성공하고 redis는
실패하면 트렌젝션 사용해서 되돌리고(rollBack) 에러 리턴
